Commit Messages

A well-formed commit message consists of a mandatory subject line, an optional body and an optional footer. The whole message should be wrapped to 72 columns.

In the next subsections, we will consider each part of a commit message in detail.

Subject Line

A subject line is intended to express a brief summary of the commit, which may be used by various developer tools and services, including (but not limiting to) the GitHub web interface and Git itself. In many cases, the only thing that is shown in a commit log being viewed by some third-party programmer will be your subject line. Therefore, you have to make it concise and abstract, letting a reader to get the high-level sense of the changes represented by this commit. Do not mention raw filenames in the subject line - use more abstract entities from the project architecture instead (for example, module names).
