General Coding Style Guidelines by Igor Gorlov

The conventions established by this document, as well as ones established by other documents in this repository, are strongly encouraged to be followed by every programmer working on my projects. In the meantime, all these rules are not dogmas: developers have the right to neglect or even ignore them in special cases.

Terminology

The following are some common terms used throughout the whole coding style documentation.

Spaceless case
A collective name for several styles of phrase writing in which whitespaces are not used.
Upper camel case
A spaceless case where each word of the phrase being written begins with a capital letter.
Example SpacesAreNotAllowedHere
Lower camel case
A spaceless case where each word of the phrase being written, except of the first one, begins with a capital letter.
Example spacesAreNotAllowedHere
Upper snake case
A spaceless case using an underscore character ("_") as a word separator and requiring all letters to be capital.
Example SPACES_ARE_NOT_ALLOWED_HERE
Lower snake case
A spaceless case using an underscore character ("_") as a word separator and totally prohibiting capital letters.
Example spaces_are_not_allowed_here
Git case
A spaceless case using hyphen character ("-") as a word separator and totally prohibiting capital letters.
Example spaces-are-not-allowed-here

Natural Language

The official language of our community is English. It should be used in almost every case where natural language is required, including the documentation, file and directory names, identifiers within the source code, commit messages, interaction between the community members, and so on. In personal communication with me, you can also use Russian.
