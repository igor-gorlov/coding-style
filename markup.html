Terminology

Natural text
Text written in a natural language and bearing the main semantic load.
Computer text
Text that is one or more of the following:
  • Written in one of programming languages.
  • Pseudocode.
  • Output of a computer program.
  • Input made by a user of a computer.
  • Either a name of a file, or a name of a directory, or a name of a VCS branch.
Exclusive emphasis technique (EET)
A method of text highlighting that is never combined with other exclusive emphasis techniques (including itself).
Code emphasis
An exclusive emphasis technique which uses a monospaced font and (if possible) semantically marks the text being emphasized as program code (for example, with <code> HTML tag).
Term introduction emphasis (TIE)
An exclusive emphasis technique that works on a short text fragment and makes it non-breaking italic.
Important fragment emphasis (IFE)
An exclusive emphasis technique that works on a short text fragment and makes it non-breaking bold.
Emphasis conflict
A situation when more than 1 exclusive emphasis technique should theoretically be applied to the same word or phrase.

Partition

  • Begin a document with a title in the form of a top-level header (that is, a header of level 1). There must be exactly one top-level header in each individual document.
  • Split large documents into logical sections with headers of level 2.
  • Split large sections into logical subsections with headers of level 3.
  • If a section or a subsection can be presented as a sequence of short statements, split it into clauses by using a (potentially multi-level) list.
  • If a section or a subsection can be presented as a sequence of statements, but these statements are too long, make regular paragraphs instead of clauses.
  • Clauses and paragraphs can be mixed up together within a section or a subsection.

Emphasis

  • Always use a proportional font for natural text.
  • Use code emphasis for all computer text.
  • Use term introduction emphasis for the first occurrence of every new term.
  • Use important fragment emphasis for the most valuable words and phrases.
  • TIE and IFE highlightings should not span long pieces of text (only critical fragments are to be emphasized) and should not appear too frequently.
    Examples
    An unacceptably long emphasis In our community, it is not allowed to insult other people or to impersonate the administrator.
    Too many relatively short highlightings In our community, it is not allowed to insult other people or to impersonate the administrator.
    The recommended level of emphasis In our community, it is not allowed to insult other people or to impersonate the administrator.
  • Any combinations of emphasis techniques, including nested highlighting, are forbidden: a single text fragment can use only single emphasizing method. If a conflict takes place, the style that is located higher in the following priority list is picked:
    1. Computer text.
    2. New term.
    3. Important text
